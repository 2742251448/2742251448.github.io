hexo博客搭建

emm... 1458 words in the article | you are the th friend who reading now

第一步：安装和注册

1.安装node.js(网址：https://nodejs.org/en/ )，选择LTS版本

2.安装git（网址：https://git-scm.com/downloads/ ），选择相应的版本

3.安装时应该注意出路径外，其余的都保持默认

4.到GitHub（网址：https://github.com/ ）注册一个账号，然后进入，点击右上角的 + ，选择 New repository，创建一个新的存储库，Repository name 一定是Owner.github.io,如我Owner是2742251448，那么我的Repository name就一定是2742251448.github.io 其它的都不变，直接点击Create Repository


第二步：打开终端（即命令提示符）

1.通过管理员打开终端

第三步：配置一个git的global身份

1.配置身份

2.检测身份是否配置成功,如果出现你配置的name和email就表示配置成功

注意：email和name的后面都有一个空格，其中name和email都要和注册GitHub的保持一致

第四步：检测node.js是否安装成功

1.检测node.js和npm (npm是和node.js一起安装的) 是否安装成功

如果出现版本型号就表示配置成功

第五步：安装cnpm

1.安装cnpm

2.检测是否安装成功

如果出现版本型号就表示配置成功

3.借助cnpm安装hexo

第六步：建立一个新的文件夹

1.新建一个文件夹

2.转到新建的文件夹位置(假设为 F:\桌面\blog)，先转到盘符，然后再转到文件夹位置

第七步：在新建的文件夹中进行博客的初始化并检测

1.初始化(耗时较长，需耐心等待)

2.启动hexo

出现http://localhost:4000 就表示成功，把这个地址复制到浏览器的地址栏中并打开，这时会出现hexo页面。到这里就表示成功了一半。

3.Ctrl + C 停止，Y

第八步：部署到GitHub上

1.部署到GitHub上

第九步：设置_config.yml文件

1.通过记事本或其它应用打开新建文件下的_config.yml

2.找到type，添加文字。 type: git ; repo: 我们之前建立的存储库地址；branch: master

冒号后面一定要有空格，如果_config.yml文件中没有repo和branch,添加即可

第十步：上传

1.上传

2.如果出现name和email,填我们之前配置git的global身份的name和email

3.如果出现password,这表示我们需要输入密码，我们只需要输入GitHub的密码即可，在输入的过程中，密码不会显示出来，我们尽管输入即可

做到这里博客的搭建就已经完成了，下篇文章讲如何换博客的主题
